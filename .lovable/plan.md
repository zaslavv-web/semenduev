## Проблема

На `/admin` сначала загружается сессия, ставится `loading = false`, и страница рендерится. Запрос роли из `user_roles` ещё выполняется, а `isAdmin` по умолчанию `false` → пользователь видит «Доступ запрещён», хотя он админ. Особенно заметно после редиректа со страницы сброса пароля.

В БД роль `admin` для пользователя есть — проблема чисто на клиенте.

## Фикс

В `src/routes/admin.tsx` (компонент `AdminPage`):

1. Добавить отдельное состояние `roleLoading`, инициализировать `true`.
2. В эффекте проверки роли:
   - при отсутствии `userId` → `setRoleLoading(false)`;
   - перед запросом → `setRoleLoading(true)`;
   - после ответа (или ошибки) → `setRoleLoading(false)` и `setIsAdmin(!!data)`.
3. Условие показа экрана загрузки: `loading || (userId && roleLoading)` — не рендерить «Доступ запрещён», пока роль ещё проверяется.
4. Также показать `toast.error(error.message)` при ошибке запроса (для диагностики, если RLS заблокирует).

## Файлы

- `src/routes/admin.tsx` — добавить `roleLoading`, объединить условие загрузки, обработать ошибку запроса роли.

Других изменений не требуется. БД, RLS и `/reset-password` трогать не нужно.
