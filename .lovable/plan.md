## План доработок сайта

### 1. Персонализированный hero
Сгенерировать новое фоновое изображение через нейросеть в текущей цветовой гамме (тёмно-синий, золотой акцент) с фигурой делового мужчины-фронтмена (нашего героя) в деловом стиле. Заменить `hero.backgroundImage` в defaults.

**Уточнение:** для генерации правдоподобного «нашего фронтмена» нужны референсы. Если фото нет — сгенерируем обобщённый образ (мужчина 35-45, деловой костюм, уверенный) в той же стилистике.

### 2. Сокращение дублирования: объединить «Этапы работы» + «Инструменты»
- Блок **«Подход» (HowIWork)** — оставить как есть (короткий смысловой).
- Блок **«Этапы работы» (Process)** — расширить, добавив в каждый шаг секцию «Инструменты этапа» (что именно используется на этом шаге). Это объясняет «что происходит после заявки».
- Удалить отдельный блок **Tools** из главной страницы и из навигации.

### 3. Расширить кейсы до 6
В `cases.items` добавить 3 новых типичных ситуации (кассовый разрыв, текучка ключевых сотрудников, падение маржи, конфликт с партнёром, потеря крупного клиента, неконтролируемый рост долгов).

### 4. Переименовать блок «Чек-лист»
Заменить заголовок «Бесплатный материал» / «Чек-лист» на **«Антикризисный чек-лист для собственника»** + подзаголовок «Материал для первичной диагностики бизнеса». Обновить тексты CTA, ссылающиеся на «чек-лист».

### 5. Дипломы — единый премиальный вид
- Привести карточки к единому формату: одинаковая рамка, тень, соотношение сторон (3/4), внутренний padding, фон.
- Добавить под каждым дипломом подпись-категорию: MBA / Экономические науки / Профильная специализация / Антикризисное управление.
- Расширить структуру `diplomas.items` полем `category`.

### 6. Политика конфиденциальности и юридический блок
- Создать страницу `/privacy` с текстом политики конфиденциальности и согласия на обработку ПДн (шаблонный текст под РФ).
- В Footer добавить ссылку «Политика конфиденциальности» и блок реквизитов (плейсхолдер «ИП будет указано позднее» — редактируется через CMS).
- Под каждой формой (Contact, Checklist) добавить мелким шрифтом: «Нажимая кнопку, вы соглашаетесь с [политикой конфиденциальности](/privacy)».

### 7. Разнообразить CTA и усилить эмоциональный/срочностный посыл
- В `CTA` компоненте сделать лейблы кнопок настраиваемыми (через props), чередовать формулировки: «Разобрать ситуацию», «Получить диагностику», «Обсудить ситуацию», «Получить первую помощь».
- Добавить под формами строку срочности: «Каждый месяц без изменений усиливает потери бизнеса».
- В Hero / About / Process вставить сильные смысловые цитаты: «Бизнес не рушится за один день», «Кассовый разрыв начинается задолго до нехватки денег», «Падение прибыли — симптом, а не проблема».

### Технические изменения
- `src/lib/content/defaults.ts` — расширить схемы (process.steps[].tools, diplomas.items[].category, footer.legal, checklist.title/subtitle, формы.consentText, формы.urgencyText), удалить tools из defaults опционально (оставить, чтобы не ломать БД, но не рендерить).
- `src/components/site/Process.tsx` — рендер инструментов внутри шага.
- `src/components/site/Cases.tsx` — поддержка 6 карточек (уже динамический).
- `src/components/site/Diplomas.tsx` — единая стилизация + category.
- `src/components/site/Footer.tsx` — ссылка на политику + реквизиты.
- `src/components/site/Contact.tsx`, `Checklist.tsx` — consent + urgency.
- `src/components/site/CTA.tsx` — настраиваемые лейблы кнопок.
- `src/components/site/Hero.tsx` — новый фон + смысловая строка.
- `src/routes/index.tsx` — убрать `<Tools />`.
- `src/routes/privacy.tsx` — новая страница.
- `src/routes/admin.tsx` — поля для новых сущностей (privacy редактируемо, footer.legal, формы consent/urgency, diplomas.category, process.tools).
- Сгенерировать новое hero-изображение в `public/`.

- Конкретное фото фронтмена — без референса генерируется обобщённый образ в той же стилистике.
